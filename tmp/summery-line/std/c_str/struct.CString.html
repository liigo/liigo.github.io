The representation of a C String.

This structure wraps a *libc::c_char, and will automatically free the memory it is pointing to when it goes out of scope.

Methods

unsafe fn new(buf: *const i8, owns_buffer: bool) -> CString

Create a C String from a pointer, with memory managed by C's allocator API, so avoid calling it with a pointer to memory managed by Rust's allocator API, as the behaviour would not be well defined.

Failure

Fails if buf is null

fn as_ptr(&self) -> *const i8

Return a pointer to the NUL-terminated string data.

.as_ptr returns an internal pointer into the CString, and may be invalidated when the CString falls out of scope (the destructor will run, freeing the allocation if there is one).

fn main() { let foo = "some string"; // right let x = foo.to_c_str(); let p = x.as_ptr(); // wrong (the CString will be freed, invalidating `p`) let p = foo.to_c_str().as_ptr(); }
let foo = "some string";

// right
let x = foo.to_c_str();
let p = x.as_ptr();

// wrong (the CString will be freed, invalidating `p`)
let p = foo.to_c_str().as_ptr();

Example

extern crate libc; fn main() { let c_str = "foo bar".to_c_str(); unsafe { libc::puts(c_str.as_ptr()); } }
extern crate libc;

fn main() {
    let c_str = "foo bar".to_c_str();
    unsafe {
        libc::puts(c_str.as_ptr());
    }
}

unsafe fn unwrap(self) -> *const i8

Unwraps the wrapped *libc::c_char from the CString wrapper.

Any ownership of the buffer by the CString wrapper is forgotten, meaning that the backing allocation of this CString is not automatically freed if it owns the allocation. In this case, a user of .unwrap() should ensure the allocation is freed, to avoid leaking memory. You should use libc's memory allocator in this case.

Prefer .as_ptr() when just retrieving a pointer to the string data, as that does not relinquish ownership.
